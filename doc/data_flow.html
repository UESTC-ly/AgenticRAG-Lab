Stage · Retriever

HybridRetriever

src/agentic_rag_lab/retrieval/hybrid.py
对每条 sub_query，召回 top_k=5，按 doc_id 去重合并进 evidence_pool（保留更高分）。

Lexical both

retrieval/lexical.py

BM25-like 词频打分，top_k×2 召回（有 reranker 时扩成更大候选池）。

Semantic rule

retrieval/semantic.py

离线代理 baseline：token 扩展 + cosine-like，相当于“轻量伪语义检索”。

Embedding Semantic real

retrieval/embedding_semantic.py

真实 dense retrieval：源码默认 BAAI/bge-m3；benchmark 文档另收录 bge-small-en-v1.5 实验。文档 embedding 会先预计算，再做向量相似度。

RRF Fusion both

text.py :: reciprocal_rank_fusion

1 / (60 + rank)，用名次融合 lexical / semantic 两路结果，避免直接比较不同分数量纲。

Light Rerank rule

retrieval/hybrid.py

规则版不接真实 reranker，而是用 token overlap bonus 做轻量重排。

Cross-Encoder Rerank real

retrieval/reranker.py

真实版用 cross-encoder 对候选做精排；源码默认 BAAI/bge-reranker-v2-m3，benchmark 文档另收录 bge-reranker-base 实验。

list[EvidenceItem] → 合并进 evidence_pool
Stage · Critic

EvidenceCritic

src/agentic_rag_lab/critic.py :: evaluate(query, evidence)
判断证据是否充足。当前实现主要看 query token 覆盖率 + 教育/作者/国籍等领域信号；不足时产出 follow_up_queries。
CritiqueResult { is_sufficient, follow_up_queries, confidence }
Decision

sufficient? → 跳出循环  |  not sufficient? → active_queries ← follow_up_queries

max_iterations=3 是硬上限，防止 critic 无限追问死循环。
evidence_pool.values() → list[EvidenceItem]
⑤ 答案合成
Stage · Synthesizer

Synthesizer

synthesizer.py（rule） · llm_synthesizer.py（real）

CitationSynthesizer rule

按 score 排序 → 抽高命中句子 → 拼接答案；yes/no 题和教育类问题有专门规则，并直接返回 doc_id citations。

LLMSynthesizer real

通过 Ollama 调本地模型；源码默认 qwen2.5:7b，也可改成其他本地模型。先把证据编号为 [1][2]...，再把模型输出里的编号反向映射回 doc_id，并做 citation 校验。

tuple[str, list[str]] = (answer, citations)
⑥ 最终返回
Output · RunResult

RunResult { query, answer, citations, route, iterations, trace }

一次完整调用的最终产物。
answer：如 "Exeter College, Oxford"
citations：如 ["hobbit-author", "tolkien-oxford"]
iterations：实际跑了几轮
trace：list[TraceEvent]，给调试 / Web UI / `trace` 子命令展示
为什么这个架构值得看？
主循环（agent.py）不关心任何组件实现，只关心数据契约。所以把规则版 semantic 换成真实 dense retriever，不需要改主循环；把规则版 synthesizer 换成 LLM 版，也只是构造函数里换实例。
两条路径怎么走？
· 离线路径（build_demo_agent()）：rule 组件拼起来，零依赖，可完全离线运行，用于开发 / CI / baseline。
· 真实路径（build_real_agent()）：semantic / reranker / synthesizer 切到 real 组件，需要 sentence-transformers + 本地 ollama serve。
· 任一真实组件失败都能独立降级，不会让整条 loop 挂掉。
